# Everything Doubao Code 配置说明文档

## 项目概述

本项目集成了开源的 "Everything Doubao Code" 配置,为 Trae IDE 提供了一套完整的 AI 辅助开发工具集。这套配置汇集了丰富的实战经验,涵盖了 SubAgents、Skills、Commands、Rules、Hooks 等模块。

## 配置结构

```
.trae/
├── rules/          # 规则集 - 定义开发规范和工作流程
├── skills/         # 技能 - 专业知识库和工作流程
├── commands/       # 命令 - 快捷操作指令
├── hooks/          # 钩子 - 自动化触发器
└── agents/         # 代理 - 专业子代理(预留)
```

## 模块说明

规则集定义了开发过程中必须遵循的规范和流程。

- 命名规范
- 代码结构要求
- 错误处理规范
- 注释规范
- 性能考虑
- 安全规范
- 测试要求
- 文档要求
- 版本控制规范

**[development-workflow.md](.trae/rules/development-workflow.md)**
- 需求分析阶段
- 设计阶段
- 开发阶段
- 测试阶段
- 部署阶段
- 维护阶段
- 文档管理
- 团队协作

**[security.md](.trae/rules/security.md)**
- 密码和认证
- 数据保护
- 输入验证
- API 安全
- 错误处理
- 依赖管理

- 应急响应

#### 使用方式

规则集会自动应用于所有开发任务,确保代码质量和安全性符合标准。

### 2. Skills(技能)

技能是专业知识库,为 AI 提供特定领域的知识和工作流程。

#### 已配置的技能

**[code-review.md](.trae/skills/code-review.md)**
- 全面的代码审查能力
- 多维度审查(质量、安全、性能、可维护性)
- 结构化的审查流程
- 优先级分类的问题报告
- 最佳实践验证

**[debugging.md](.trae/skills/debugging.md)**
- 系统化的调试方法论
- 问题定义和信息收集
- 假设形成和验证
- 常见问题类型识别
- 调试工具使用指南
- 最佳实践和检查清单

**[testing.md](.trae/skills/testing.md)**
- 全面的测试策略
- 多种测试类型(单元、集成、端到端、性能、安全)
- 测试设计原则(FIRST、AAA、测试金字塔)
- 测试覆盖率分析
- 测试自动化和持续集成

#### 使用方式

技能会根据任务类型自动激活,提供专业的知识和指导。

### 3. Commands(命令)

命令是快捷操作指令,可以快速执行常见任务。

#### 已配置的命令

**[review.md](.trae/commands/review.md)**
- 执行全面的代码审查
- 支持指定文件或目录
- 生成详细的审查报告
- 按优先级分类问题

使用示例:
```
/review
/review src/
/review src/utils/helper.py
```

**[test.md](.trae/commands/test.md)**
- 执行测试套件
- 支持多种测试类型
- 生成测试覆盖率报告
- 支持监视模式

使用示例:
```
/test
/test --unit
/test --coverage
/test --watch
```

#### 使用方式

在对话中直接使用命令,系统会自动识别并执行相应的操作。

### 4. Hooks(钩子)

钩子是自动化触发器,在特定事件发生时自动执行预定义的操作。

#### 已配置的钩子

**[pre-commit-review.md](.trae/hooks/pre-commit-review.md)**
- 在代码提交前自动执行代码审查
- 检查代码质量、安全性、测试和文档
- 生成审查报告
- 阻止不符合标准的代码提交

**[memory-persistence.md](.trae/hooks/memory-persistence.md)**
- 自动保存和恢复会话上下文
- 实现跨会话的记忆持久化
- 分类存储项目、工作、知识和用户记忆
- 支持记忆搜索和关联

#### 使用方式

钩子会自动在相应的事件触发时执行,无需手动干预。

## 使用指南

### 日常开发流程

1. **开始新任务**
   - 系统自动恢复项目上下文
   - 查看当前工作进度和待办事项

2. **编写代码**
   - 遵循代码质量规范
   - 参考开发流程规范
   - 注意安全规范要求

3. **代码审查**
   - 使用 `/review` 命令进行代码审查
   - 根据审查报告修复问题
   - 提交前会自动执行预提交审查

4. **测试验证**
   - 使用 `/test` 命令运行测试
   - 查看测试覆盖率报告
   - 修复失败的测试

5. **提交代码**
   - 预提交钩子自动检查代码质量
   - 确保所有检查通过
   - 提交代码

6. **会话结束**
   - 系统自动保存工作进度
   - 记录重要决策和解决方案
   - 下次会话自动恢复

### 高级使用技巧

1. **自定义配置**
   - 根据项目需求调整规则集
   - 添加项目特定的技能
   - 创建自定义命令

2. **集成 CI/CD**
   - 将代码审查集成到 CI 流程
   - 在 PR 时自动运行测试
   - 生成质量报告

3. **团队协作**
   - 共享配置文件到团队
   - 统一开发规范
   - 建立代码审查流程

## 最佳实践

1. **遵循规范**
   - 严格遵循代码质量规范
   - 按照开发流程执行任务
   - 注意安全规范要求

2. **充分利用技能**
   - 让 AI 使用专业技能完成任务
   - 参考技能中的最佳实践
   - 学习技能中的知识

3. **使用命令**
   - 使用快捷命令提高效率
   - 定期执行代码审查
   - 持续运行测试

4. **利用钩子**
   - 让钩子自动执行检查
   - 依赖记忆持久化保持上下文
   - 减少手动操作

## 常见问题

### Q: 如何禁用某个规则?
A: 可以在规则文件中设置 `enabled: false` 来禁用特定规则。

### Q: 如何添加新的技能?
A: 在 `.trae/skills/` 目录下创建新的 Markdown 文件,按照技能模板编写内容。

### Q: 如何自定义命令?
A: 在 `.trae/commands/` 目录下创建新的 Markdown 文件,定义命令的功能和使用方式。

### Q: 钩子会影响性能吗?
A: 钩子会增加一些执行时间,但可以显著提高代码质量。可以通过配置优化钩子的执行。

### Q: 如何备份配置?
A: 将 `.trae/` 目录添加到版本控制,定期备份配置文件。

### Q: 如何更新配置?
A: 定期检查 Everything Doubao Code 项目的更新,手动同步新的配置。
