# 删除 span 文字下划线的修改方案

## 问题分析

用户想要删除 `span` 文字内容的下划线。通过代码分析，我发现：

### 目标元素
```javascript
<span className="text-xs md:text-sm tracking-wide font-['Space_Grotesk'] pb-0.5 transition-colors group-hover:border-opacity-100" style={{
  color: 'var(--hero-accent-orange)',
  borderBottom: '2px solid var(--hero-accent-orange)',
  borderColor: 'var(--hero-accent-orange)'
}}>
  向下滚动查看完整案例
</span>
```

### 下划线来源
下划线是通过 `borderBottom` 样式属性设置的，而不是 `text-decoration` 属性。

## 解决方案

### 修改方案
移除 `borderBottom` 和 `borderColor` 样式属性，保留其他样式。

### 修改后代码
```javascript
<span className="text-xs md:text-sm tracking-wide font-['Space_Grotesk'] pb-0.5 transition-colors group-hover:border-opacity-100" style={{
  color: 'var(--hero-accent-orange)'
}}>
  向下滚动查看完整案例
</span>
```

## 技术说明

1. **修改位置**：`src/components/sections/HeroSection.tsx` 文件中的第 186-190 行
2. **修改内容**：删除 `borderBottom` 和 `borderColor` 样式属性
3. **影响范围**：仅影响向下滚动提示的文字样式，不影响其他元素
4. **兼容性**：修改后在所有设备上都不会显示下划线

## 预期效果

- ✅ 文字保持原有颜色（橙色）
- ✅ 文字大小和字体保持不变
- ✅ 移除文字下方的下划线
- ✅ 保持悬停效果的过渡动画
- ✅ 所有设备端显示一致

## 风险评估

- **风险等级**：低
- **影响范围**：仅影响单个样式属性
- **回滚方案**：如需恢复下划线，只需重新添加 `borderBottom` 样式即可

## 执行步骤

1. 打开 `src/components/sections/HeroSection.tsx` 文件
2. 找到向下滚动提示的 `span` 元素
3. 删除 `borderBottom` 和 `borderColor` 样式属性
4. 保存文件并验证修改效果
