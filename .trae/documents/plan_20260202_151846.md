## 接入 Everything Claude Code 配置方案

### 方案概述

由于 Trae IDE 不支持 Claude Code CLI 的插件市场命令,需要手动集成 Everything Claude Code 项目。

### 实施步骤

**步骤1: 下载 Everything Claude Code 项目**

* 从 GitHub 克隆或下载: <https://github.com/affaan-m/everything-claude-code>

* 解压到本地临时目录

**步骤2: 分析项目结构**

* 查看项目的目录结构,了解各个模块的组织方式

* 识别可以集成到 Trae IDE 的配置文件

**步骤3: 选择性集成配置**
根据你的需求选择以下模块进行集成:

* **SubAgents**: 专业子代理配置(规划、架构、代码审查等)

* **Skills**: 工作流技能定义

* **Commands**: 斜杠命令配置

* **Rules**: 代码规范规则集

* **Hooks**: 自动化钩子配置

**步骤4: 配置到 Trae IDE**

* **4.1 配置文件复制**
  * 在项目根目录创建 `.trae` 文件夹
  * 复制以下目录结构到 `.trae` 文件夹:
    * `agents/` - 子代理配置文件
    * `commands/` - 斜杠命令配置
    * `skills/` - 工作流技能定义
    * `hooks/` - 钩子配置和脚本
    * `rules/` - 代码规范规则
    * `scripts/` - 辅助脚本文件

* **4.2 环境适配和调整**
  * **路径适配**:
    * 将所有 `${CLAUDE_PLUGIN_ROOT}` 变量替换为绝对Windows路径
    * 确保脚本路径使用Windows风格的反斜杠 `\`
  * **模型配置**:
    * 将所有代理配置中的 `model: opus` 替换为 `model: doubao`
    * 移除所有对Anthropic和Claude的引用
  * **Windows兼容性**:
    * 禁用或修改tmux相关的钩子配置
    * 确保所有脚本使用Windows兼容的命令格式
  * **依赖检查**:
    * 确保Node.js已安装并添加到系统PATH
    * 验证脚本文件的执行权限

* **4.3 配置验证**
  * 重启Trae IDE以加载新配置
  * 验证代理配置是否正确加载
  * 测试斜杠命令是否可用
  * 检查钩子脚本是否正常执行
  * 验证模型配置是否使用doubao

**步骤5: 创建说明文档**

* **5.1 集成模块记录**
  * 详细记录已集成的所有模块
  * 包含每个模块的功能说明和配置位置
  * 提供模块间的依赖关系图

* **5.2 使用指南和注意事项**
  * 提供详细的使用步骤和示例
  * 说明Windows环境下的特殊操作
  * 列出常见问题和解决方案
  * 提供最佳实践建议

* **5.3 配置更新日志**
  * 记录每次配置变更的内容和原因
  * 包含版本号和更新日期
  * 维护变更历史记录

### 预期成果

* 获得专业的子代理支持,提升开发效率

* 自动化的代码规范检查

* 跨会话的记忆持久化能力

* 丰富的快捷命令和工作流
