# 代码审查命令

## 命令描述

执行全面的代码审查,检查代码质量、安全性、性能和可维护性。

## 使用方法

```
/review [文件路径或目录]
```

## 参数说明

- `文件路径或目录`: 可选,指定要审查的文件或目录。如果不提供,则审查当前工作目录。

## 功能特性

1. **代码质量检查**
   - 检查代码可读性
   - 验证命名规范
   - 分析代码复杂度
   - 检测重复代码

2. **安全检查**
   - 识别 SQL 注入风险
   - 检测 XSS 攻击风险
   - 验证 CSRF 保护
   - 检查敏感信息泄露

3. **性能分析**
   - 评估算法复杂度
   - 识别性能瓶颈
   - 检查数据库查询
   - 分析资源使用

4. **最佳实践验证**
   - 检查设计模式应用
   - 验证 SOLID 原则
   - 检查错误处理
   - 验证日志记录

## 输出格式

```
# 代码审查报告

## 总体评价
- 代码质量评分: X/10
- 主要优点: ...
- 主要问题: ...

## 严重问题
1. [问题描述]
   - 位置: 文件:行号
   - 风险: ...
   - 建议: ...

## 高优先级问题
...

## 中优先级问题
...

## 低优先级建议
...

## 最佳实践建议
...
```

## 示例

```
/review
/review src/
/review src/utils/helper.py
```

## 注意事项

- 确保代码已提交到版本控制
- 大型项目可能需要较长时间
- 建议在 Pull Request 前执行
- 可以与 CI/CD 流程集成
